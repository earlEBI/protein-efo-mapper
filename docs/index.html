Analyte Mapper

A curator-first mapper for analytes and traits to ontology terms. Designed for transparent, auditable GWAS metadata curation.

Offline-first setup UKB-aware trait mapping ICD10 / PheCode routing MONDO-in-EFO QC flag

Quick Start

python3 -m venv .venv
source .venv/bin/activate
	python -m pip install -U pip
		python -m pip install -r requirements.txt
		python -m pip install -e .
		analyte-efo-mapper setup-bundled-caches
		analyte-efo-mapper cache-status --strict --output-json final_output/analyte_mapper_cache_status.json

setup-bundled-caches auto-downloads missing official UKB metadata, refreshes MONDO ICD10 mappings, builds the ICD10 label cache when missing, and rebuilds the local index. If live MONDO refresh fails (for example offline), setup falls back to the local MONDO cache.

analyte-efo-mapper map --input data/analytes.tsv --output final_output/analytes_efo.tsv
	analyte-efo-mapper trait-map --input data/traits.tsv --output final_output/traits_mapped.tsv --review-output final_output/traits_review.tsv

Mapping Priority

Analyte mode (map):

  1. Exact analyte cache
  2. UniProt/metabolite identity resolution
  3. Exact ontology label/synonym
  4. Lexical/token fallback
  5. Review queue for non-validated rows

Trait mode (trait-map):

  1. Cache exact ICD10 and supplemental ICD10
  2. Cache exact PheCode
  3. Cache exact UKB field and supplemental UKB field
  4. Cache exact/fuzzy text
  5. efo.obo exact ICD10 xref and exact label/synonym
  6. efo.obo fuzzy fallback
  7. UKB category fallback

Column GroupColumns
Input Trace input_row_id, input_query, negation, input_type, input_trait_scale, input_source_code, input_source_data_type, input_icd10, input_icd10_label, input_phecode

QC Flags validation, qc_review_flag, term_not_in_efo, mondo_missing_ids
Audit Text evidence, provenance

negation is yes when input contains cues such as none, no, not, without, or none of the above. Negation rows are mapped as review_required when a candidate exists; non-informative options like none of the above stay not_mapped. input_trait_scale applies branch preference: quantitative favors measurement terms, binary favors non-measurement terms.

Important: if a MONDO mapping is not imported in EFO and no better active EFO-family term is available, term_not_in_efo is set to term not in EFO and the row is review-required.

Refresh Trait Cache From New Studies TSV

analyte-efo-mapper trait-cache-refresh \
  --studies-tsv /path/to/gwas-catalog-studies.tsv

This refresh writes QC artifacts and updates cache rows using this policy:

  1. High-confidence ontology resolution
  2. Unambiguous mapping per normalized trait
  3. Specific label overlap filter
  4. Skip conflicts with existing curated cache

A refresh state file tracks the last processed studies hash. If unchanged, refresh is skipped unless --force-refresh is set.

Compiled index: measurement_index.json with term_meta, exact_term_index, token_index, analyte_index, accession_alias_index, gene_symbol_accession_index, optional gene_id_accession_index when the UniProt alias cache includes gene_ids, metabolite_alias_index, and metabolite_concept_index. It is generated locally during setup rather than relied on as a tracked Git artifact.

Curator QC Routine

  1. Run trait-map with --review-output.
  2. Start with rows where qc_review_flag=yes.
  3. Prioritize rows where term_not_in_efo is populated.
  4. Use matched_on, source_file, evidence, and provenance to decide.
  5. After decisions, rerun setup-bundled-caches to rebuild supplements/index.
