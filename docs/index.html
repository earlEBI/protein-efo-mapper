Start Here

Run the mapper locally with a browser UI.

This tool lets you upload a file, choose either trait mapping or analyte mapping, and download TSV results from a local Uvicorn web app. You do not need to learn the CLI first.

Trait mapper Protein mapper Metabolite mapper TSV outputs

Fastest path

If you just want the app running, follow these four commands from the repository root.

python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
.venv/bin/pip install -e .
.venv/bin/python -m uvicorn --app-dir skills/pqtl-measurement-mapper/web app:app --reload --host 127.0.0.1 --port 8000

Then open http://127.0.0.1:8000 in your browser.

What The App Does

Trait Mapper

Use this for disease traits, phenotype labels, ICD10-style text, PheCodes, and UKB field-based trait inputs.

Protein / Metabolite Mapper

Use this for UniProt accessions, gene symbols, protein names, HMDB IDs, ChEBI IDs, and metabolite names.

Download Results

The app writes TSV outputs. Trait jobs can also write a Catalog bulk-add TSV when final mapped terms are still missing from the local Catalog export.

Step-By-Step Setup

1

Clone the repository

mkdir -p ~/src
cd ~/src
git clone https://github.com/earlEBI/analyte-efo-mapper.git
cd analyte-efo-mapper
2

Create a Python environment

python3 -m venv .venv
.venv/bin/pip install -U pip
.venv/bin/pip install -r requirements.txt
.venv/bin/pip install -e .
3

Prepare bundled caches

analyte-efo-mapper setup-bundled-caches

This builds or reuses the local reference files the mapper needs, including the measurement index and bundled trait resources.

4

Start the Uvicorn web app

.venv/bin/python -m uvicorn --app-dir skills/pqtl-measurement-mapper/web app:app --reload --host 127.0.0.1 --port 8000

Open http://127.0.0.1:8000 and upload your file.

If your local checkout uses skills-src

.venv/bin/python -m uvicorn --app-dir skills-src/pqtl-measurement-mapper/web app:app --reload --host 127.0.0.1 --port 8000

Which Mode Should I Choose?

Choose trait-map if your file contains

  • Disease names or phenotype labels
  • ICD10 terms or union-like ICD strings
  • PheCodes
  • UKB field-based trait text
query	trait_scale	code	data_type
Hand grip strength (left)	quantitative	46	UKB Data Field
Union#M2571#M25.71 Osteophyte (Shoulder region)	binary	Union	ICD10

Choose map if your file contains

  • UniProt accessions
  • Gene symbols
  • Protein names
  • HMDB or ChEBI IDs
  • Metabolite names
query	input_type
Q9ULI3	accession
LIPC	gene_symbol
HMDB0000122	metabolite_id
glucose	metabolite_name

Using The Web App

1. Upload a file

Use a .tsv, .csv, .txt, .list, or .tab input file.

2. Pick the mode

Select map for proteins/metabolites or trait-map for traits and diseases.

3. Download results

When the job completes, use the download links for mapped TSVs, review TSVs, QC TSVs, and the Catalog bulk-add TSV when present.

The app is local. Your files stay on your machine unless you choose to move or share them yourself.

What The Main Outputs Mean

Output What it is for
mapped.tsv or traits_mapped.tsv Your main results file with the chosen ontology mapping for each input row.
unmapped.tsv Rows the analyte mapper could not confidently map.
traits_review.tsv Trait rows that deserve manual review because the match is uncertain, negated, or otherwise flagged.
traits_qc_risk.tsv Trait rows with stronger QC concerns, such as ontology or specificity issues.
traits_catalog_bulk_add.tsv Trait terms that still are not in the local Catalog export after exact existing-term checks, ready for bulk add review.

Trait Mapping Notes

Helpful columns

For the best trait-mapper results, include query and, when available, trait_scale, code, and data_type.

Review behavior

Negated or ambiguous rows can be mapped but still sent to review. That is expected and helps keep QC visible.

Catalog-aware output

The mapper now checks proposed terms against the local Catalog export, prefers existing Catalog terms when possible, and can emit a bulk-add TSV for any exact unresolved terms.

TSV-only output

The current workflow writes TSV artifacts rather than duplicating both TSV and CSV exports.

If You Prefer The CLI

Analytes

analyte-efo-mapper map \
  --input data/analytes.tsv \
  --output final_output/analytes_efo.tsv

Traits

analyte-efo-mapper trait-map \
  --input data/traits.tsv \
  --output final_output/traits_mapped.tsv \
  --review-output final_output/traits_review.tsv
