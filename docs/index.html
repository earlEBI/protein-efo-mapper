1. Setup Details (Novice)

Do this once on a new machine. Follow the exact steps below even if you have never used Python before.

  1. Open your command line: use Terminal (Mac/Linux) or PowerShell (Windows).
  2. Install Git and Python 3.10+: if missing, install from git-scm.com/downloads and python.org/downloads. Then check versions: git --version, python3 --version (Mac/Linux), or py --version (Windows).
  3. Copy one full setup block below and run it: this downloads the project, creates an isolated Python environment, installs required packages, creates folders, and copies your input template to data/analytes.tsv.
If a command says not found, stop there, install the missing tool (Git or Python), then run the block again.

Mac/Linux: run this exactly

git clone https://github.com/earlEBI/analyte-efo-mapper.git
cd analyte-efo-mapper
python3 -m venv .venv
source .venv/bin/activate
python -m pip install -U pip
python -m pip install -r requirements.txt
python -m pip install -e .
mkdir -p data final_output
cp docs/input_template.tsv data/analytes.tsv
analyte-efo-mapper setup-bundled-caches

Windows PowerShell: run this exactly

cd analyte-efo-mapper

Next: edit data/analytes.tsv with your own IDs (UniProt, gene symbol, gene ID, protein name, or metabolite IDs/names).
The final command in each setup block validates bundled protein/metabolite/trait caches and builds the local index without internet downloads.
Trait mode requires skills/pqtl-measurement-mapper/references/efo.obo. If missing, setup/trait-map can auto-provision it from your pinned bundled URL (recommended: repo release asset efo.obo.gz).
To use your own release asset, pass --efo-obo-bundled-url https://github.com/<owner>/<repo>/releases/latest/download/efo.obo.gz.
Setup uses a lightweight UniProt profile by default (reviewed-like + requires gene symbol) and writes skills/pqtl-measurement-mapper/references/uniprot_aliases_light.tsv.
If you need the full bundled UniProt cache for setup/indexing, run: analyte-efo-mapper setup-bundled-caches --uniprot-profile full
If you plan to map many gene_id inputs, run once after setup: analyte-efo-mapper uniprot-alias-enrich --uniprot-aliases skills/pqtl-measurement-mapper/references/uniprot_aliases_light.tsv --workers 8

3. Copy-Paste Commands

Choose one command based on what you are mapping.

a) Proteins

analyte-efo-mapper map \
  --input data/analytes.tsv \
  --output final_output/analytes_efo.tsv \
  --withheld-triage-output final_output/withheld_for_review_triage.tsv \
  --index skills/pqtl-measurement-mapper/references/measurement_index.json \
  --entity-type protein \
  --workers 8 \
  --parallel-mode process \
  --progress

b) Metabolites

analyte-efo-mapper map \
  --input data/analytes.tsv \
  --output final_output/analytes_efo.tsv \
  --withheld-triage-output final_output/withheld_for_review_triage.tsv \
  --index skills/pqtl-measurement-mapper/references/measurement_index.json \
  --metabolite-aliases skills/pqtl-measurement-mapper/references/metabolite_aliases.tsv \
  --entity-type metabolite \
  --workers 8 \
  --parallel-mode process \
  --progress

c) Traits

analyte-efo-mapper trait-map \
  --input data/traits.tsv \
  --output final_output/traits_mapped.tsv \
  --trait-cache skills/pqtl-measurement-mapper/references/trait_mapping_cache.tsv \
  --efo-obo skills/pqtl-measurement-mapper/references/efo.obo \
  --min-score 0.82 \
  --review-output final_output/traits_review.tsv \
  --progress

Trait cache QC is automatic: cache IDs are checked against efo.obo, obsolete IDs are remapped where possible, and unresolved IDs are warned and excluded.

How Mapping Works

Deterministic identity is always preferred; lexical fallback is used only when needed.

1) Cache sources used by runtime

  • uniprot_aliases.tsv (bundled source) -> uniprot_aliases_light.tsv (generated default for speed).
  • metabolite_aliases.tsv (bundled HMDB-derived aliases, optionally rebuilt from pinned HMDB SDF).
  • trait_mapping_cache.tsv (bundled curated trait cache).
  • efo_measurement_terms_cache.tsv (bundled measurement terms, refreshable via refresh-efo-cache).
  • efo.obo (full ontology for trait fallback + obsolete-ID checks; auto-provisioned if missing).
  • measurement_index.json (local compiled index from setup).
Measurement cache and full efo.obo are intentionally separate: measurement cache is runtime-optimized for analyte mapping; full OBO is for trait mapping and ontology QC.

2) Mapping order and method

  • Protein/metabolite map: resolve IDs first (UniProt/HMDB/ChEBI/KEGG), then exact/synonym term hits, then token/lexical fallback, then validation gates.
  • Trait trait-map: cache exact ICD10 -> cache exact PheCode -> cache exact text -> cache fuzzy text (if no ontology-exact) -> exact efo.obo -> fuzzy efo.obo.

3) Validation

  • Identity checks: accession/concept consistency blocks near-name collisions.
  • Context checks: default blood context allows blood/plasma/unlabeled and excludes serum unless added.
  • Safeguards: ratio terms require ratio-like query intent; weak lexical-only matches are withheld.
  • Trait cache QC: cache ontology IDs are checked against efo.obo; obsolete IDs are remapped where possible and unresolved IDs are excluded from auto output.

4) Withheld triage

  • Enable with --withheld-triage-output final_output/withheld_for_review_triage.tsv.
  • Rows are classified as reject_high, review_needed, accept_medium, or accept_high.
  • Includes query label and suggested mapped label/subject for quick manual QC.

4. Additional Options

Use these only when you need context tuning, cache refresh, or deeper review outputs.

Measurement Context and Optional Arguments

Use --measurement-context to control tissue/matrix filtering and --additional-contexts to opt into extras.

  • --measurement-context blood (default): allows blood + plasma + unlabeled terms, excludes serum.
  • --measurement-context plasma: plasma-only context.
  • --measurement-context blood --additional-contexts serum: blood/plasma plus serum.
  • --measurement-context cerebrospinal_fluid: CSF-only context.
  • --additional-context-keywords <phrase>: free-text add-on (for example aorta, adipose tissue).
analyte-efo-mapper map \
  --input data/analytes.tsv \
  --output final_output/analytes_efo.tsv \
  --index skills/pqtl-measurement-mapper/references/measurement_index.json \
  --measurement-context plasma \
  --workers 8 \
  --parallel-mode process \
  --progress
analyte-efo-mapper map \
  --input data/analytes.tsv \
  --output final_output/analytes_efo.tsv \
  --index skills/pqtl-measurement-mapper/references/measurement_index.json \
  --measurement-context auto \
  --additional-context-keywords aorta \
  --workers 8 \
  --parallel-mode process \
  --progress
analyte-efo-mapper map \
  --input data/analytes.tsv \
  --output final_output/analytes_efo.tsv \
  --review-queue-output final_output/review_queue.tsv \
  --index skills/pqtl-measurement-mapper/references/measurement_index.json \
  --entity-type auto \
  --workers 8 \
  --parallel-mode process \
  --progress
analyte-efo-mapper map \
  --input data/analytes.tsv \
  --output final_output/analytes_efo.tsv \
  --withheld-triage-output final_output/withheld_for_review_triage.tsv \
  --index skills/pqtl-measurement-mapper/references/measurement_index.json \
  --entity-type auto \
  --workers 8 \
  --parallel-mode process \
  --progress
Triage works for both proteins and metabolites. Protein rows use accession/gene-symbol consistency; metabolite rows use HMDB/ChEBI/KEGG ID/xref consistency.
  • Useful optional flags: --top-k, --min-score, --workers, --name-mode, --entity-type, --auto-enrich-uniprot, --metabolite-aliases, --unmapped-output, --review-queue-output, --withheld-triage-output.
--auto-enrich-uniprot is incremental: it fetches missing aliases only for accession-like queries in your current input, writes them to the file set by --uniprot-aliases, then rebuilds the index. It is not a full UniProt rebuild. If you are using lightweight setup, pass --uniprot-aliases skills/pqtl-measurement-mapper/references/uniprot_aliases_light.tsv.
For full gene-ID coverage in your local lightweight UniProt cache, run: analyte-efo-mapper uniprot-alias-enrich --uniprot-aliases skills/pqtl-measurement-mapper/references/uniprot_aliases_light.tsv --workers 8 (add --refresh-all to refresh every accession row).

Outputs

  • final_output/analytes_efo.tsv: main mappings with confidence and validation.
  • final_output/withheld_for_review_triage.tsv (optional): triaged withheld rows to prioritize manual review.
  • final_output/review_queue.tsv (optional): broader manual-review file with closest unresolved candidates; useful suggestions, but many are likely incorrect.
  • final_output/analytes_unmapped.tsv: optional pure unresolved list.
This website is documentation and templates. Mapping runs in your local Python environment.

Refresh EFO Cache (Optional)

Use this when you want to refresh the full EFO/OBA measurement cache and rebuild the local index. Rebuild is on by default and should usually stay on.

analyte-efo-mapper refresh-efo-cache \
  --download-url https://github.com/EBISPOT/efo/releases/latest/download/efo.obo \
  --term-cache skills/pqtl-measurement-mapper/references/efo_measurement_terms_cache.tsv \
  --index skills/pqtl-measurement-mapper/references/measurement_index.json \
  --analyte-cache skills/pqtl-measurement-mapper/references/analyte_to_efo_cache.tsv \
  --uniprot-aliases skills/pqtl-measurement-mapper/references/uniprot_aliases.tsv \
  --metabolite-aliases skills/pqtl-measurement-mapper/references/metabolite_aliases.tsv
No local OBO file is required. If --efo-obo is not provided, the script downloads from the URL above.

Troubleshooting

  • python3: command not found: install Python and restart terminal.
  • ModuleNotFoundError: activate virtual environment and run python -m pip install -r requirements.txt.
